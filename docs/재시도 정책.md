# 재시도 정책

## 목적
- 트래픽 처리 경로별 재시도 동작을 일관된 기준으로 운영한다.
- 재시도 대기 방식, 최대 시도 횟수, 예외 처리 기준을 명시한다.

## 공통 원칙
- 재시도는 실패 직후 대기(`sleep`) 후 수행한다.
- 대기는 지수 백오프를 사용한다.
- 기본 백오프 값(base)은 `50ms`이며, 경로별 설정으로 주입한다.
- 재시도는 동기 처리로 수행한다(`Thread.sleep` 기반 유지).
- 재시도 가능 예외(`retryable`)만 재시도한다.
- 재시도 불가능 예외는 즉시 중단하고 상위 실패 처리로 전이한다.

## 지수 백오프 규칙
- 공식: `delay = baseBackoffMs << (retryAttempt - 1)`
- 예시(base=50ms):
  - 재시도 1회차: `50ms`
  - 재시도 2회차: `100ms`
  - 재시도 3회차: `200ms`
  - 재시도 4회차: `400ms`
  - 재시도 5회차: `800ms`

## 경로별 최대 시도 횟수
- 정책 write-through 재시도:
  - `초기 1회 + 재시도 3회` (총 4회 시도)
- DB claim 재시도:
  - `초기 1회 + 재시도 3회` (총 4회 시도)
- **hydrate 재시도:**
  - `초기 1회 + 재시도 5회` (총 6회 시도)
- Refill 재시도:
  - `초기 1회 + 재시도 3회` (총 4회 시도)
- Redis 멱등키 제거 재시도:
  - `초기 1회 + 재시도 3회` (총 4회 시도)

## 정책 해석 주의사항
- 모든 즉시 재시도가 3회로 통일된 것은 아니다.
- hydrate 경로는 별도 정책으로 재시도 5회를 사용한다.

## 구현 체크리스트
- 재시도 루프에 실패 후 `sleep` 위치가 일관적인지 확인한다.
- 경로별 최대 시도 횟수가 정책과 일치하는지 확인한다.
- 예외 분류가 retryable/non-retryable로 명확히 분리되는지 확인한다.
- 인터럽트(`InterruptedException`) 발생 시 `Thread.currentThread().interrupt()`를 호출하는지 확인한다.

## 테스트 체크리스트
- 초기 시도 실패 후 재시도 성공 케이스
- 재시도 최대 횟수 소진 후 실패 케이스
- 비재시도 예외 즉시 중단 케이스
- 재시도 대기(backoff) 계산 검증 케이스
